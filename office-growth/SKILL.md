# office-growth — 個人成長儀表板產生器

## 這個 skill 在做什麼

把使用者「想變強但不知道怎麼追蹤」的模糊焦慮,轉成一份**可量化、可每週檢視**的成長儀表板,並視需要搭配一份**季度學習計畫**。核心心法:成長不是靠感覺,是靠「看得見的進步」證明;而學習不是收藏知識,是要有明確的輸出目標。

## 何時使用

使用者出現以下情況時主動套用:
- 想建立自己的成長/學習追蹤系統
- 覺得自己一直在學新東西,但說不出自己到底變強在哪
- 想規劃某一季/某段時間要主修什麼技能
- 開了很多線上課程、買了很多書,但沒有真的學完或用上
- 要求「幫我做一個成長儀表板」「幫我規劃學習計畫」這類請求

---

## 第一步:先問診——為什麼想學這個?

在動手做儀表板前,若使用者的目標還很模糊(例如只說「我想變厲害」),先問這三題,不要直接跳去產出模板：

1. **我為什麼要學這件事?**(工作需要?想轉職?補技能?單純興趣?)
2. **我想在多久之內達到什麼程度?**(能開口講?能產出作品?還是只求理解?)
3. **我現在的生活狀態,每週可以誠實投入多少時間?**(不要抓理想時數)

如果使用者已經很清楚自己要練什麼、要多久,可以跳過這步直接進第二步。

---

## 第二步:成長儀表板三步驟設計

**步驟一、列出 3 個成長主題**(不要多,但要具體聚焦——「我要變更好」不算)

常見三個範疇可參考:
- 工作技能(如資料整理、簡報表達、專案管理)
- 職場軟實力(如溝通協作、時間管理、跨部門合作)
- 個人成長(如閱讀吸收、邏輯思考、抗壓能力)

**步驟二、為每個主題設定可量化的小指標**(不必是死板 KPI,但要「可追蹤」)

例如:
- 資料整理 → 這週有沒有設計出一個新的資料框架?(有/沒有)
- 時間管理 → 一週內準時交付任務的比例?(％)
- 閱讀吸收 → 本月完成幾篇有做筆記的閱讀?(篇數)

**步驟三、設一個每週更新的檢視時間**(建議 15 分鐘):打勾推進的指標、打叉停滯的指標,並設定下週要加強哪一塊。

### 產出模板(直接套用使用者的主題填入)

```
## [使用者姓名/暱稱] 的成長儀表板 — 第 __ 週

| 成長主題 | 本週目標 | 完成情況 | 調整方向 | 備註 |
|---|---|---|---|---|
| 主題一 | | ☐ 已完成 ☐ 未完成 | | |
| 主題二 | | ☐ 已完成 ☐ 未完成 | | |
| 主題三 | | ☐ 已完成 ☐ 未完成 | | |

下週小挑戰:
```

**產出時**:依使用者實際描述的目標填入 3 個主題與對應指標,不要用預設的資料整理/時間管理/閱讀當萬用範例硬套——除非使用者的情境剛好貼合。

---

## 第三步(選用):季度學習計畫四步驟

當使用者要規劃的是一段時間(例如一季)的學習方向,而不只是每週追蹤時,加上這一層:

**步驟一、訂出明確學習目標**(要看得見的成果,不是模糊幻想)
- 例:「我要學會 Notion 並打造自己的工作看板」而不是「我想變厲害」

**步驟二、拆成三層學習模組**(像打電動一樣分關卡)
- 基礎理解(Lv.1):核心概念、基礎知識
- 操作應用(Lv.2):要動手做的實作、可用的模板
- 進階優化(Lv.3):做到一定程度後要強化的細節

**步驟三、安排時間節奏**(避免每晚學到半夜、三天就放棄)
- 一週安排 2-3 個學習時段(每次 30-60 分鐘)
- 排出每週一個 Review Day 複習整理
- 每月安排一次小產出(寫一篇筆記、錄一段教學、完成一個 side project)

**步驟四、設計回饋機制**,讓自己知道「真的有在進步」
- 成果追蹤表(完成幾堂、幾份作業、幾段輸出)
- 學習日誌(每次學完寫一句「今天我學到⋯⋯」)
- 找人分享學到的東西(教別人是最強複習法)

### 學習提醒 check-list(產出季度計畫時附上)

- [ ] 我知道我為什麼學這個
- [ ] 我知道什麼時間我能固定投入
- [ ] 我把大目標拆成幾個小任務了
- [ ] 我有設計「每週可交付成果」
- [ ] 我有設計「複習＆回顧機制」
- [ ] 我不是為了學而學,而是為了變得更有用

### 「每季一到兩個主修」原則

呼應成長儀表板的聚焦精神:每一季只設定 1-2 個「主修課題」,其餘技能先擱著。不要每週換主題——累積不起來的學習只是知識觀光。每學一樣東西,都設一個對應的「輸出任務」（例如:學簡報 → 做一份幫主管做決策的提案;學數據分析 → 幫部門做一次數據回顧),學完沒產出等於沒學。

---

## 產出時的通用原則

- 主題與指標必須貼合使用者實際講的目標,不要套用範例當真實輸出。
- 指標要「可追蹤」但不需要嚴苛精確,重點是讓使用者看得到自己的行動頻率。
- 若使用者要的是找工作/轉職方向的規劃(而非技能追蹤),提醒這個 skill 專注在「量化追蹤與學習節奏」,職涯方向選擇是更大的話題,可以另外討論但不勉強套框架。
- 保持模板簡潔可重複使用——每週檢視只需要 15 分鐘,不要把儀表板設計得太複雜導致使用者懶得更新。
